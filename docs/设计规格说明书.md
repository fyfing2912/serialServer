## 2. config.json 数据结构

```json
{
  "ports": [
    {
      "port": 4001,
      "enabled": true,
      "serial_params": {
        "baud_rate": 9600,
        "data_bits": 8,
        "stop_bits": "1",
        "parity": "None"
      },
      "response_rules": [
        { "type": "exact",   "match": "AT\r",      "reply": "OK\r\n" },
        { "type": "contains","match": "version",   "reply": "simulator v1.0\r\n" },
        { "type": "wildcard","match": "show *",    "reply": "not supported\r\n" }
      ]
    }
  ],
  "service_running": false
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `port` | int | 监听端口号，必须唯一 |
| `enabled` | bool | 是否启用 |
| `serial_params.baud_rate` | int | 波特率 |
| `serial_params.data_bits` | int | 数据位（5/6/7/8） |
| `serial_params.stop_bits` | string | 停止位（"1"/"1.5"/"2"） |
| `serial_params.parity` | string | 校验位（"None"/"Odd"/"Even"/"Mark"/"Space"） |
| `response_rules` | array | 有序规则列表 |
| `response_rules[].type` | string | exact / contains / wildcard |
| `response_rules[].match` | string | 匹配模式（支持转义符） |
| `response_rules[].reply` | string | 响应内容（支持转义符） |
| `service_running` | bool | 全局服务运行状态 |

#### 3.2.3 Telnet 状态机

```
状态：NORMAL / IAC / SB / SB_COM_PORT
```

**字节流解析逻辑：**

| 当前状态 | 收到字节 | 动作 |
|---------|----------|------|
| NORMAL | 0xFF (IAC) | 切换到 IAC |
| NORMAL | 其他 | 送入规则引擎，回传响应 |
| IAC | 0xFF | 转义：原样发送 0xFF |
| IAC | 0xFB (WILL) | 记录，等待选项字节 |
| IAC | 0xFC (WONT) | 处理 WONT |
| IAC | 0xFD (DO) | 记录，等待选项字节 |
| IAC | 0xFE (DONT) | 处理 DONT |
| IAC | 0xFA (SB) | 切换到 SB，记录选项 |
| IAC | 0xF0 (SE) | SB 结束，切换回 NORMAL |
| IAC | 其他 | IAC 协议命令，切换回 NORMAL |
| SB | COM_PORT_CMD (0x00) | 切换到 SB_COM_PORT，收集子协商数据 |
| SB | 其他 | 跳过子协商，等待 SE |
| SB_COM_PORT | 0xFF | 切换到 IAC（SB 内 IAC 转义） |
| SB_COM_PORT | 0xF0 (SE) | 解析子协商，切换回 NORMAL |
| SB_COM_PORT | 其他 | 追加到子协商缓冲区 |

#### 3.2.4 RFC2217 子协商解析

**SETBAUD (1)：** 4 字节 uint32，大端序

```
IAC SB COM_PORT_CMD SETBAUD [B3][B2][B1][B0] IAC SE
```

**SETDATASIZE (2)：** 1 字节

```
IAC SB COM_PORT_CMD SETDATASIZE [N] IAC SE
N ∈ {5, 6, 7, 8}
```

**SETPARITY (3)：** 1 字节

```
IAC SB COM_PORT_CMD SETPARITY [P] IAC SE
P: 0=None, 1=Odd, 2=Even, 3=Mark, 4=Space
```

**SETSTOPSIZE (4)：** 1 字节

```
IAC SB COM_PORT_CMD SETSTOPSIZE [S] IAC SE
S: 1=1位, 2=1.5位, 3=2位
```

**SETCONTROL (5)：** 1 字节（解析并确认）

```
IAC SB COM_PORT_CMD SETCONTROL [C] IAC SE
```

**SETFLOW (6)：** 1 字节（解析并确认）

```
IAC SB COM_PORT_CMD SETFLOW [F] IAC SE
```

**SUSPEND (7)：** 无参数（解析并确认）

```
IAC SB COM_PORT_CMD SUSPEND IAC SE
```

#### 3.2.5 规则引擎

```python
def match_rule(data: bytes, rules: list[dict]) -> bytes:
    """
    按顺序匹配规则，返回第一个匹配的 reply
    若无匹配，返回原数据（回显）
    """
    text = data.decode('utf-8', errors='replace')
    for rule in rules:
        if rule['type'] == 'exact':
            if text == unescape(rule['match']):
                return unescape(rule['reply'])
        elif rule['type'] == 'contains':
            if unescape(rule['match']) in text:
                return unescape(rule['reply'])
        elif rule['type'] == 'wildcard':
            pattern = unescape(rule['match']).replace('*', '')
            if text.startswith(pattern):
                return unescape(rule['reply'])
    return data  # 默认回显
```

**unescape 实现：**

```python
def unescape(s: str) -> str:
    s = s.replace('\\r', '\r')
    s = s.replace('\\n', '\n')
    s = s.replace('\\\\', '\\')
    return s
```

**职责：** FastAPI 应用入口，REST API 实现，静态文件托管。

#### 3.3.1 路由设计

```
GET  /              →  static/index.html
GET  /api/ports     →  返回所有端口配置
POST /api/ports     →  新增端口

POST /api/service/start →  启动所有已启用端口
POST /api/service/stop  →  停止所有端口
```

#### 3.3.2 API 响应格式

```json
{ "success": true,  "message": "操作描述" }
{ "success": false, "message": "错误原因" }
```

- 新增端口时，检查端口号是否已存在
- 端口号重复 → 返回 400

## 5. 关键实现细节

### 5.1 规则优先级

规则列表有序，**从上到下**优先级递减。实现时保持 JSON 数组顺序不变。

### 5.2 通配符匹配算法

```python
# wildcard: 仅末尾 *，匹配零个或多个任意字符
def wildcard_match(text: str, pattern: str) -> bool:
    # pattern 形如 "show *"
    prefix = pattern.rstrip(' *')  # "show "
    if not pattern.endswith(' *'):
        return False
    return text.startswith(prefix)  # 前缀匹配即可
```

### 5.3 RFC2217 回应策略

服务端对所有收到的子协商进行**镜像回应**（echo back），确保客户端确认参数已被服务端接受。

例外：SETCONTROL/SETFLOW/SUSPEND 仅确认，不执行实际操作。

### 5.4 端口变更流程（PUT）

```
1. 解析请求体，提取除 port 外的所有字段
2. 若请求体包含 port 字段且值 ≠ path 中的 {port} → 400
3. 若启用了新状态为 true → restart_port(port)
4. 保存配置
5. 返回成功
```

### 5.5 服务重启时的端口冲突

- stop_port 时完全释放端口
- start_port 时重新 bind
- 若端口被占用，asyncio 会抛出异常 → 捕获后记录日志，返回失败

## 8. 异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 客户端异常断开 | 记录日志，不影响服务，继续接受其他连接 |
| 端口被占用 | 日志报错，返回 API 失败，不崩溃 |
| 配置 JSON 损坏 | 读取失败时使用默认空配置，写回修复 |
| RFC2217 非法子协商 | 忽略，不回应，避免死循环 |
| Telnet 状态机异常字节序列 | 重置到 NORMAL 状态 |
