# 软件串口服务器（串口服务器模拟软件）需求规格说明书

## 1. 概述

### 1.1 项目背景
作为研发测试用的软件版串口服务器，模拟真实串口服务器的 TCP Server + Telnet(RFC2217) 功能，用于客户端（如 PuTTY、带外网管系统）的对接测试。

### 1.2 目标
仿真串口服务器的 TCP Server + Telnet(RFC2217) 功能，允许客户端通过 Telnet 协议连接指定端口，完成：
- RFC2217 串口参数协商（被动响应模式）
- 串口数据透传 + 可配置的指令响应（规则引擎）

### 1.3 范围边界（不做）
| 功能 | 状态 |
|------|------|
| 多连接支持（每个端口仅 1 个 TCP 客户端） | ❌ 不做 |
| 空闲超时 | ❌ 不做 |
| WebSocket 日志 | ❌ 不做 |
| 用户登录/权限 | ❌ 不做 |
| Web 实时连接数 | ❌ 不做 |
| PoE / IO / Modbus | ❌ 不做 |
| 主动推送 RFC2217 | ❌ 不做 |

---

## 2. 协议实现规范

### 2.1 Telnet 基础
- 处理 IAC (0xFF) 转义
- 支持的基本选项：

| 选项名 | 值 | 协商行为 |
|--------|-----|----------|
| ECHO | 0x01 | DO/DONT |
| SUPPRESS GO AHEAD | 0x03 | WILL/WONT |
| NAWS | 0x1F | 可不实现 |

### 2.2 RFC2217 关键常数

| 名称 | 值（十进制） | 说明 |
|------|-------------|------|
| COM_PORT_OPTION | 33 | Telnet 选项号（用于 IAC DO/WILL） |
| COM_PORT_CMD | 0 | 子选项头（用于 IAC SB ... SE） |

### 2.3 子选项类型（完整列表）

| 命令 | 值 | MVP 处理策略 |
|------|---|-------------|
| SETBAUD | 1 | 解析并保存（无范围限制） |
| SETDATASIZE | 2 | 解析并保存（仅校验 5/6/7/8） |
| SETPARITY | 3 | 解析并保存（N/O/E/M/S 选其一） |
| SETSTOPSIZE | 4 | 解析并保存（1/1.5/2） |
| SETCONTROL | 5 | 解析、确认，不实际控制硬件 |
| SETFLOW | 6 | 解析、确认，不执行流控 |
| SUSPEND | 7 | 解析、确认，不暂停数据流 |

### 2.4 协商行为

**Telnet 选项协商：**
- 收到 `WILL COM_PORT_OPTION` → 支持，回复 `DO COM_PORT_OPTION`
- 收到 `DO COM_PORT_OPTION` → 支持，回复 `WILL COM_PORT_OPTION`
- 收到 `WONT / DONT` → 不做特殊处理

**子协商流程（被动响应）：**
1. 客户端发送 `IAC SB COM_PORT_OPTION SETBAUD ... IAC SE`
2. 服务端解析参数，保存到内部虚拟串口设备
3. 服务端回应 `IAC SB COM_PORT_OPTION SETBAUD ... IAC SE`（确认）

**说明：** 服务端不主动推送参数变更。

### 2.5 数据透传

- TCP 收到的非 IAC 序列数据作为"串口输入"送入规则引擎
- 规则引擎生成的响应数据作为"串口输出"原样通过 TCP 连接发送
- 不需要对应用层数据进行行缓冲或转义

## 3. 串口仿真规则引擎

### 3.1 匹配模式

`type` 字段取值：`exact` | `contains` | `wildcard`

| 类型 | 说明 | 示例 |
|------|------|------|
| exact | 完全相等 | `"match": "AT\r"` |
| contains | 子串包含 | `"match": "version"` |
| wildcard | 通配符匹配 | `"match": "show *"` |

**通配符 `*` 规则：**
- 仅允许在模式字符串的末尾
- 匹配零个或多个任意字符（包括空格）
- 示例：`"show *"` 可匹配 `"show"`、`"show version"`、`"show port 1"`

### 3.2 转义处理

用户输入的 match / reply 字符串支持以下转义符：

| 转义符 | 实际字节 |
|--------|----------|
| `\r` | CR (0x0D) |
| `\n` | LF (0x0A) |
| `\\` | 反斜杠 `\` |

UI 中应提示用户使用上述转义符。

### 3.3 匹配顺序

- 每个端口维护有序规则列表（从上到下）
- 第一条匹配的规则生效，执行对应的 reply
- 若没有任何规则匹配，默认行为为回显（将收到的原样返回）

### 3.4 规则存储（JSON）

```json
"response_rules": [
  { "type": "exact",   "match": "AT\r",                   "reply": "OK\r\n" },
  { "type": "contains","match": "version",                 "reply": "simulator v1.0\r\n" },
  { "type": "wildcard","match": "show *",                  "reply": "not supported\r\n" }
]
```

## 4. 端口能力

| 参数 | 数值 |
|------|------|
| 最小支持端口数 | 4 |
| 推荐支持端口数 | 8 |
| 上限 | 不超过 64 |
| 每个端口监听 TCP 端口 | 用户自定义（如 4001~4064） |
| 串口参数 | 支持配置（存储，不实际控制硬件） |

## 5. Web UI 极简设计

### 5.1 端口列表页面

```
+------+----------+------------------+------+------------------------+
|  # | 监听端口 | 串口参数 | 启用 | 操作 |
+------+----------+------------------+------+------------------------+
|  1 | 4001     | 9600/8/1/N       |  √   | 编辑 删除 查看规则 |
|  2 | 4002     | 115200/8/1/N     |  ×   | 编辑 删除 查看规则 |
+------+----------+------------------+------+------------------------+
[ + 新增端口 ] [ 启动全部服务 ] [ 停止全部服务 ]
```

### 5.2 编辑端口（含规则）

**表单字段：**
- 监听端口（整数，不可重复）
- 启用（checkbox）
- 波特率（下拉：9600/19200/38400/57600/115200/230400/460800/921600）
- 数据位（5/6/7/8）
- 停止位（1/1.5/2）
- 校验位（None/Odd/Even/Mark/Space）

**响应规则（动态表格，支持增删改行）：**

| 类型（下拉） | 匹配模式 | 响应内容 | 操作 |
|-------------|----------|----------|------|
| 精确/包含/通配符 | 字符串 | 字符串 | 删除本行 |

按钮：**保存并应用**（重启该端口监听）、**取消**

### 5.3 全局控制

- 启动全部服务 / 停止全部服务（调用后端 API）

## 6. 前后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/ports | 返回所有端口配置（无实时状态） |
| POST | /api/ports | 新增端口，自动启动监听（若启用） |
| PUT | /api/ports/{port} | 修改端口，path中{port}为当前监听端口号 |
| DELETE | /api/ports/{port} | 删除端口，停止监听 |
| POST | /api/service/start | 启动所有已启用端口的监听 |
| POST | /api/service/stop | 停止所有监听 |

**响应格式：** 统一 JSON，含 `success` 和 `message`，错误时返回 4xx/5xx。

**PUT 特殊规则：**
- 若请求体中包含 `port` 字段且值 ≠ 路径中的 `{port}` → 返回 400，错误信息："监听端口号不可修改，如需更改请删除后新建"
- 其他字段修改后，若端口启用状态为 true，则重启该端口的监听器

## 7. 验收标准

1. 启动程序后，浏览器可访问 Web 界面
2. 创建端口 4001，启用，保存，后端日志显示"Listen on 4001"
3. 使用 PuTTY（Telnet 连接 127.0.0.1:4001），成功连接
4. PuTTY 输入 `AT` 回车 → 收到 `OK`（规则为精确匹配 `AT\r`）
5. 修改串口参数（如在 PuTTY 的 RFC2217 设置中改为 115200），后端日志显示接收到 SETBAUD 协商，并保存为新参数
6. 删除端口后，该端口不再可连接
7. 所有协议交互（Telnet, RFC2217）可通过后端日志验证（打印收到的 IAC 序列）

## 8. 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 后端 | Python 3.10+ / asyncio | 快速开发，asyncio 支持良好 |
| Web 框架 | FastAPI | 自动文档方便测试，支持静态文件托管 |
| 前端 | 单 HTML + Bootstrap 5 + 原生 JS | 无需构建工具，简单 |
| RFC2217 | 自研（基于 asyncio 协议解析） | 轻量，完全可控 |
| 配置存储 | 本地 config.json | 无依赖 |
| 部署 | `python main.py` 直接运行 | 或 pyinstaller 打包单文件 |
